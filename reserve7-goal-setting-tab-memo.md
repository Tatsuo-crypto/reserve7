# Reserve7「目標設定」タブ 再設計依頼メモ（Sonnet下調べ・Fable5への相談）

対象: `admin/diet-plan`の「目標設定」タブ（J-1で4タブ構成になった際、このタブだけ現状維持としていた箇所）。オーナーから複数の改善要望が出たため、ページ全体の設計をFable5に相談したい。

---

## 0. 更新履歴（今回の追記）

オーナーからの追加指示を受け、**「基礎情報からカロリー計算」は目標設定の履歴が0件（初回）のときだけ表示し、既に目標設定済みの会員では非表示にする**、という対応を先行して実装済み。これにより旧2.4「完成度・使いやすさが分からない」（および4章5番の相談事項）は解消済みのため、本メモから除外し、残る3点（2.1・2.2・2.3）についてあらためてFable5に相談したい。

## 1. 現状の構成（上から順）

1. **基礎情報からカロリー計算**: 年齢・身長・体重・活動量から基礎代謝／維持カロリー／減量スタートの3つを算出するカード。「維持カロリー」「減量スタート」には「目標に反映」ボタンがあり、押すと目標カロリーに直接反映される。**（実装済み）目標設定の履歴が0件のときのみ表示。既に目標設定がある会員には表示しない**
2. **現在の目標設定 / 新規プランの作成**（トグルで切替）:
   - 「現在の目標設定」: 目標摂取カロリー（大きく表示、読み取り専用）＋P/F/C/糖質/食物繊維/塩分の6項目（±ボタンで増減）＋生活習慣目標（水分/歩数/筋トレ/睡眠）＋保存ボタン
   - 「新規プランの作成」: 開始日＋P/F/C（3項目のみ、±ボタン）＋保存ボタン。**目標カロリーの数値表示が無い**（後述4章）
3. **目標設定の推移**: 「目標カロリー」棒グラフ（単独）と「PFC目標（kcal換算）」積み上げ棒グラフ（P/F/Cをkcal換算して積み上げ、ヘッダーに合計kcalとP/F/Cのg数を表示）の2つが並ぶ
4. **目標設定の履歴**: 開始日・カロリー・P/F/C・操作（編集/削除アイコン）の表形式リスト。「新しいプランを作成」ボタンもここにある

## 2. オーナーの要望

### 2.1 「目標設定の推移」の2グラフを1つに統合

「目標カロリー」棒グラフと「PFC目標」積み上げグラフは実質同じ情報（PFCグラフはヘッダーに既に合計kcalを表示している）。**PFC積み上げグラフ側にカロリーの表記を追加するだけで済むはず**であり、単独のカロリー棒グラフは不要ではないか、という所感。

### 2.2 「目標設定の履歴」（表）を廃止し、グラフをタップ編集に

数値の一覧表としてここに履歴を残さなくても、上部のPFC積み上げグラフ（2.1でカロリー表記を追加したもの）の**各バーをクリック/タップしたら、その期間の目標を編集できる**ようにすればよいのではないか、という提案。現状の編集/削除アイコン付きテーブルをグラフ操作に置き換えるイメージ。

### 2.3 「新しいプランを作成」の分かりにくさ

新規プラン作成の導線・フォームが分かりづらいので改善したい（具体的な違和感の内容までは言及なし。3章のSonnet所感を参照）。

## 3. Sonnetの所感（実装調査から見えた懸念点）

- **目標カロリーが常に読み取り専用**: 「現在の目標設定」「新規プランの作成」のどちらの画面でも、目標カロリーの数値を直接編集する入力欄が無い。カロリーはP/F/Cの±ボタンを操作した結果として自動計算される（`calories = protein×4 + fat×9 + carbs×4`）。「基礎情報からカロリー計算」の「目標に反映」ボタンだけがカロリーを直接書き換えるが、その後P/F/Cのどれか1つでも±すると、この直接指定した値は上書きされて消える。この二重の入力経路（カロリー直接指定 vs P/F/C由来の自動計算）が「分かりづらさ」（2.3）や「使いやすさへの疑問」（2.4）の一因になっている可能性がある
- **「新規プランの作成」フォームにカロリー表示が無い**: 開始日とP/F/Cの3項目しか無く、それらを操作した結果何kcalになるのかがその場で見えない（保存するまで分からない）。「現在の目標設定」画面にはある目標カロリーの大きな数値表示が、新規作成画面には無い、という非対称も分かりづらさの一因と思われる
- **糖質・食物繊維・塩分・生活習慣目標が「新規プランの作成」に無い**: 「現在の目標設定」では6項目＋生活習慣目標まで編集できるが、新規プラン作成時は3項目のみ。新しいプランを作る際にこれらは初期値のままになる（意図的な仕様か、実装の抜けかは不明）

## 4. Fable5に相談したいこと

1. 「目標設定の推移」を2.1の通り1つのグラフ（PFC積み上げ＋カロリー表記）に統合するか
2. 「目標設定の履歴」（表）を廃止し、統合後のグラフのバーをタップして編集する方式に置き換えるか。置き換える場合、削除操作もグラフ上で行うのか、別の導線を残すのか
3. カロリーの入力方式（読み取り専用の自動計算 vs 直接編集可能にする）をどちらに寄せるか。3章で指摘した「直接指定した値がP/F/C操作で上書きされる」という現状の矛盾を解消する設計
4. 「新規プランの作成」フォームの項目を「現在の目標設定」と揃える（カロリー表示、糖質/食物繊維/塩分、生活習慣目標）か、あえて簡易版のままにするか
