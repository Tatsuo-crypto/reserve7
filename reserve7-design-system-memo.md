# Reserve7 デザインシステム統一(色・アイコン・ボタン・世界観)提案メモ(Sonnet下調べ・Fable5への相談)

対象: 前回の「カラー統一」相談(`reserve7-color-unification-memo.md`)に対し、オーナーから「色だけでなく、矢印・アイコン・ボタンなど全体の世界観を統一したい。今は手作ったアプリ感・ダサさがある」とスコープが拡張された。色以外の部分も含めて構造的に調査したので、あわせてFable5に相談したい。

### 目指すゴール(オーナーからの明確な指定)

「個人が趣味で作ったアプリ」ではなく、**「プロのデザイナー・プロの開発者が作ったアプリだ」と初見で思われるレベルの完成度**を目指す、というのが今回の明確なゴール。単に色や角丸を1つに揃えるだけでなく、この基準に照らして今回洗い出した6項目それぞれが「プロ品質」と呼べる状態になっているかを判断軸にしたい。

---

## 1. 「手作り感」の正体(調査結果)

色の分裂(前回メモ参照)に加え、以下6点が構造的な原因として見つかった。

### 1.1 共通UIコンポーネントが事実上存在しない
`src/components/ui/`には`ErrorMessage.tsx`と`LoadingSpinner.tsx`の2つしかなく、**Button/Card/Badge等の基本部品が皆無**。結果として`<button>`タグが211箇所で個別に手書きされ、同じ「決定ボタン」という役割でも角丸・余白・色の組み合わせが毎回バラバラ(例: `rounded-md bg-indigo-600` / `rounded-xl bg-blue-600` / `rounded-full bg-blue-50 text-blue-600` が同じ「保存」的な意味の場所で混在)。

### 1.2 アイコンが手書きSVGで、同じ意味のアイコンでも形が揺れている
アイコンライブラリ(`@heroicons/react`)はpackage.jsonに入っているが**実際には1ファイルも使われておらず**、42ファイルがインラインSVGを手書きしている。「戻る」矢印(`M15 19l-7-7 7-7`)だけでも線の太さ(strokeWidth)が2/2.5/3の3パターン存在し、1ファイルだけ形状そのものが違う矢印を使っている箇所もある。加えて**11ファイルでは絵文字(🔴📅⚠️💡等)をアイコン代わりに使用**しており、SVGアイコンと絵文字が同じ画面に混在するケースもある。

### 1.3 角丸(border-radius)が8スケール、シャドウも8スケール、無秩序に混在
`rounded-full`〜`rounded-[2.5rem]`まで8種類の角丸が並列使用され、「カードは常にこの角丸」のような規則がない。1画面(admin/diet-plan)だけでも5パターンの角丸が混在。シャドウも`shadow-sm`〜`shadow-2xl`まで8種あり、階層(カード<モーダル<浮遊要素)に対応した使い分けが見えない。

### 1.4 太字指定がほぼ皆無
`font-normal`が617件使われている一方、`font-semibold`/`font-bold`/`font-medium`は合計18件のみ。見出し・CTAボタンも含めほぼ全てfont-normalで統一されており、視覚的な強弱(ヒエラルキー)が色とサイズだけに頼った平板な画面になっている。意図したミニマリズムというより、太字指定の付け忘れに近い状態。

### 1.5 日本語UIなのに読み込みフォントがLatin専用
`layout.tsx`で`next/font/google`の Inter(Latinサブセットのみ)を読み込んでいるが、UI文字列はほぼ全て日本語。Interは日本語グリフを持たないため、実際の表示はOSのフォールバックフォント任せになっている可能性が高く、意図したフォント統一が効いていない。

### 1.6 (前回メモの再掲)色の一元管理の仕組みがない
Tailwind configにカスタムパレット定義なし、globals.cssにも色のテーマ変数なし。ブランド色がblue/indigo/roseの3系統に分裂している。

## 2. まとめると

現状のダサさ・手作り感は、単一の原因ではなく、**「色」「アイコン」「角丸」「シャドウ」「太さ」「フォント」の6つの軸すべてで"統一ルールが存在せず、機能追加のたびに各画面が個別に手書きしてきた"ことの積み重なり**という診断になる。裏を返せば、これらを一元管理する仕組み(デザイントークン+共通UIコンポーネント)を1つ導入すれば、今後の機能追加でも同じ揺れが再発しなくなる。

## 3. Sonnetからの提案(たたき台)

1. **デザイントークンの導入**: Tailwind configまたはCSS変数で、色(前回メモの単一アクセントカラー方針)・角丸(例:カード=1種類、ボタン=1種類、モーダル=1種類の3段階に集約)・シャドウ(カード用/モーダル用の2〜3段階に集約)を一元定義する。
2. **共通UIコンポーネントの新設**: `Button`(プライマリ/セカンダリ/破壊的アクションの3バリアント程度)、`Card`、`Badge`を`src/components/ui/`に追加し、既存の手書きボタン・カードを段階的に置き換える。
3. **アイコンをライブラリに統一**: 既にpackage.jsonにある`@heroicons/react`(または`lucide-react`)を正式採用し、手書きSVG・絵文字アイコンを1セットのアイコンに置き換える。特に「戻る矢印」「開閉シェブロン」など頻出アイコンから着手。
4. **タイポグラフィの整理**: 見出し/本文/補足テキストの3段階程度にfont-weightとサイズの組み合わせを固定化する(例: 見出し=font-semibold、本文=font-normal、補足=font-normal+text-xs+gray)。日本語表示に対応したフォント設定(Noto Sans JP等、またはシステムフォント明示)も検討する。

## 4. Fable5に相談したいこと

1. 前回のカラー統一方針に加え、上記6項目(色/アイコン/角丸/シャドウ/太さ/フォント)を含めた「デザインシステムの再構築」として一括で方針を決めるか、それとも色→アイコン→ボタン、のように段階を分けて順番に決めていくか
2. デザイントークン+共通UIコンポーネント(Button/Card/Badge)を新設する方針そのものへの賛否。導入する場合、最初に手をつけるべきは何か(全画面共通で使用頻度の高いButtonからか、それとも一番目につくダイエットモジュール画面からか)
3. アイコンをライブラリ(heroicons/lucide-react)に統一する場合、既存の手書きSVGを一括置換するか、新規実装分から適用して既存分は徐々に置き換えるか
4. 目指す世界観の方向性(ミニマル/温かみ/信頼感重視のプロフェッショナル感、など)を一言で定義するとしたら何か。これが決まると、角丸の丸み具合・太字の使い所・アイコンの線の太さなど、今回洗い出した各要素の具体的な数値(rounded-xlにするかrounded-2xlにするか等)を迷わず決められる
5. 改修の優先順位: 前回メモの「販売用パッケージ化」「テナント分離」の話とこのデザイン統一は同時並行で進めるべきか、デザイン統一を先に片付けてから販売準備に進むべきか
6. **「プロが作ったアプリだ」と思われる完成度を明確な達成基準にする場合、何をもって"プロ品質"と判定するか。** 具体的には、参考にすべき既存の著名アプリ・デザインシステム(例: 特定のヘルスケア/フィットネス系アプリ、Appleの人間工学ガイドライン、Material Design等)をベンチマークとして指定するか、それとも今回洗い出した6項目(色1色化・アイコン統一・角丸/シャドウの階層化・太字によるヒエラルキー・日本語対応フォント・共通コンポーネント化)を全て満たせば十分と考えるか。オーナーの主観的な「ダサさ」判定に頼らず完成度を検証できる基準を、この相談を通じて言語化したい
