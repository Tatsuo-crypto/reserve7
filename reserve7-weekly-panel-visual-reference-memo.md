# Reserve7 アプリ全体のデザイン参考画像の相談メモ(Sonnet下調べ・Fable5への相談)

## 0. スコープの確認(重要)

**この相談はダイエットモジュールに閉じた話ではなく、reserve7アプリ全体(会員側・管理者側の両方)のデザイン言語についての相談である。** N章で確定した色・アイコン・角丸・シャドウ・太さ・フォントのトークン(N-4)はもともとアプリ全体を対象に決定されたものであり、PR-N3で先行実装したのは「まずダイエットモジュールから着手する」というN-2の優先順位づけに従った結果に過ぎない。今回オーナーから提示された2つの参考画像も、ダイエット画面に限定した要望ではなく、**色の使い方・タイポグラフィ・レイアウトパターンといったデザイン言語そのものへの示唆**として扱う。ここで出る結論(配色方針・タイポグラフィ・カードの型など)は、ダイエットモジュールだけでなく予約・ホーム・設定・管理者の各画面にも今後(PR-N4以降)適用されていく前提で回答してほしい。

対象: N章(デザインシステム統一)のPR-N1〜N3実装後、オーナーから2つの参考画像が提示された。
1つ目は黒ベース・緑基調の食事記録アプリの「帳尻合わせ」週間画面(1章)。「全体のデザイン(文字サイズ・記号のサイズ感・色合い)はこれを参考にしたい」「特にダイエットの1週間のグラフページはこれを参考にしてほしい」「色合いは白ベースでオレンジ・緑にしてほしい」という要望。
2つ目はMyFitnessPal(画面下部ナビの「MFコーチ」表記で確認)の複数画面(5章)。「見やすかった。このアプリの良いところを参考にしたい」という要望。
**1つ目の要望はN章で確定したばかりの単一アクセントカラー(blue系)方針、およびA-4/H-2で確定した「進捗バーに善悪の色をつけない」方針と正面から矛盾する**ため、2つの参考画像をあわせてFable5に相談したい。

具体的な適用先の第一弾として引き続きダイエットの週間パネル(会員側・管理者側の両方、3章参照)を挙げているが、これは「最初に手を入れる画面」であって「対象範囲そのもの」ではない点に注意。

## 1. 参考画像の客観的な観察

### 1.1 全体構成(黒ベース・アプリ名不明、いわゆる食事記録アプリの週間ビュー)
- ヘッダー: 緑のグラデーションバー、白抜きの戻る矢印・情報アイコン、中央タイトル「帳尻合わせ」
- 週切替: 左右の丸型矢印ボタン+中央に「2026年7月6日(月)〜7月12日(日)」という期間表示
- 「記録チェック表」: 7日ぶんの丸型チェックマーク(緑の縁取り+緑のチェック=記録あり、グレーの縁取り+「＋」=未記録)を横並びで表示。日付(大きい数字)+曜日(小さい文字)を上下に配置。今日の列だけ背景色でハイライト
- トグルスイッチ(「ビタミン・ミネラルを表示する」)が1行

### 1.2 各項目(カロリー・PFC・糖質・食物繊維・塩分)の表示形式
```
カロリー                          19,109 / 17,262kcal        +1,847kcal
[===================緑〜黄〜オレンジのグラデーションで塗られたバー===]
```
- ラベル(左、中サイズ、太字寄り)
- 数値は「実績 / 目標」を横並びでかなり大きく表示(このアプリの中で最も強調されている要素)。単位(kcal/g)は数値よりだいぶ小さく添える
- 右端に**残り/超過量**: 目標以内なら**緑**で「あと◯g」、目標を超えたら**赤**で「+◯kcal」「+◯g」
- バーは実績値の割合に応じて**緑→黄→オレンジ/赤へグラデーションで連続的に変化**しながら伸びる(単色べた塗りではない)。目標を超えた分は右端が赤くなる

### 1.3 タイポグラフィ・サイズ感
- 主要数値(実績値)が画面内で圧倒的に大きい(見出しの2〜3倍程度の視覚的重み)
- 目標値・単位は数値よりかなり小さく、明度も落として「添え物」であることを視覚的に示している
- ラベル(項目名)は中サイズだが数値ほどのインパクトはない
- 全体的に「大きい数字・太い文字・はっきりした色」で、Sonnetが以前の調査で報告した現行reserve7の「font-normalばかりで平板」とは対照的な、メリハリの強い書体設計

### 1.4 色の使われ方(最重要ポイント)
このアプリは「進捗バー・残り/超過表示の色そのもので、目標に対して良い状態か悪い状態かを常に判定して見せる」設計になっている:
- 緑 = 目標以内(安全圏)
- 黄〜オレンジ = 目標に近づいている、またはやや超過
- 赤 = 明確に目標超過(警告)
これはバーの塗りだけでなく、「あと◯g」(緑)/「+◯kcal」(赤)という数値表示自体の色分けにも一貫して使われている。

## 2. 既存決定との矛盾点(Fable5に判断してほしい核心)

### 2.1 単一アクセントカラー方針(N-4)との矛盾
N-4では「アクセント色は1色(blue系)に集約する」ことが確定し、PR-N1〜N3で実装済み。今回オーナーが希望する「白ベースでオレンジ・緑」は、**2色(オレンジ+緑)を意図的に使う**配色であり、N-4の「静かな信頼感=色数を減らす」という原則と方向性が異なる。

### 2.2 「進捗バーに善悪の色をつけない」方針(A-4/H-2)との矛盾
- A-4: 「未達の赤色・警告色をホームで使わない。達成時だけ色が付く非対称設計。未達は無色」
- H-2: 「前週比の色は中立(グレー+矢印)で統一。カロリー増=悪とは限らず、増減の善悪判断はパネルの仕事にしない」
参考画像は逆に、**目標超過を赤/オレンジで明確に「悪い」と示す**設計。これはH章で「対比え」として明示的に採用した設計思想と正反対であり、単なる配色の変更ではなく**評価方針そのものの転換**を意味する。

### 2.3 PFC色分け(既存の意味色)との整合性
現行のP=amber/F=emerald/C=blueという項目識別色は「どの項目か」を示す色であり、参考画像の「目標に対して良いか悪いか」を示す色とは軸が別物。両方を同じバーに載せる場合、「項目識別色」と「達成度色」のどちらを優先するかを決める必要がある(参考画像は達成度色のみで項目識別色を使っていない=カロリーもPFCも塩分も同じ緑〜赤のグラデーションルールで統一されている)。

## 3. 対象画面の特定(第一弾の適用先。0章のとおりこれが範囲の全てではない)

オーナーの「ダイエットの1週間のグラフページ」は、直近のH〜J〜M章で作り込んだ以下の画面群を指すと考えられる:
- 会員側: 記録タブ内の週間パネル(`WeeklyProgressPanel`)・週間まとめ(`WeeklySummaryPanel`)
- 管理者側: 週間パネルタブ(同一コンポーネントを共有)
これらは既にH章で「達成率%・記録日平均・7ブロック・週合計」という情報構成が確定しており、参考画像はこの情報構成自体を否定するものではなく、**同じ情報を大きな文字・強い色でどう見せるか**という表現面の参考だと解釈できる。

なお、この画面は`WeeklyProgressPanel`/`WeeklySummaryPanel`というF-4以来の「会員側・管理者側で共有するコンポーネント」であるため、ここでの結論(色・タイポグラフィ)を変更すれば**自動的に会員側・管理者側の両方に反映される**。つまりこの画面を第一弾に選ぶこと自体が、既に「会員側・管理者側どちらか一方だけ」という限定にならない構造になっている。

## 5. 第2の参考画像(MyFitnessPal)の観察: 参考にしたい「良いUX」

オーナーが「見やすい」「良いところを参考にしたい」と挙げたのはこちらのアプリ(黒ベースだが、色の使い方の哲学ではなく**情報設計・レイアウトパターン**を参考にしたいという趣旨と解釈)。

### 5.1 「摂取量一覧」の2列カードグリッド形式
各項目(カロリー・タンパク質・脂質・炭水化物…)を同一フォーマットのカードで並べる: ラベル+期間(「今日」)→ 目標位置に目印(縦線)付きの細いプログレスバー → 大きな実績値+小さな単位 → 詳細への矢印(`>`)。全項目が同じ型なので視線が迷わない。現行reserve7は項目ごとに縦積みリスト(4行構成)だが、この2列グリッドは一覧性が高く画面を有効に使えている。

### 5.2 色分けの一貫性(項目識別色の徹底)
カロリー=青、タンパク質=オレンジ(サーモン系)、脂質=黄、炭水化物=緑、という対応が、ミニバー・カードの棒・週間PFCブロック・グラフ凡例まで**全画面で完全に統一**されている。現行reserve7のP=amber/F=emerald/C=blueとは対応関係こそ違うが、「1つの対応表をアプリ全体で徹底する」という規律そのものが参考になる(N-4のPFC色維持の方針と両立可能)。

### 5.3 ダッシュボードのミニプレビュー(スパークライン)
ホーム画面の「現状の分析」「データカレンダー」セクションで、各カードに小さな折れ線・棒グラフ・記録有無ドットのプレビューを埋め込み、タップで詳細画面に遷移する。「概要を一目→気になるものだけ深掘り」という設計で、情報を全部見せずに済んでいる。

### 5.4 詳細画面(グラフ)の構成順序
期間平均の大きな数値 → 棒グラフ(目標ラインの点線＋推奨範囲の帯を重ねて表示) → 期間切替ピル(1週間/1ヶ月/3ヶ月/6ヶ月/1年/全期間) → 色の凡例 → 説明文カード(「カロリーについて」等)。「目標ライン」と「推奨範囲」を別々に見せている点が現行reserve7(目標値のみ)より情報が豊か。

### 5.5 記録継続の可視化(30日ドット)
「データカレンダー」で直近30日分を小さな四角のドット列で表示し、記録がある日だけ色付き(体重=緑、摂取量=青)。現行の7ブロックより長い期間を一目で見渡せる方式。

### 5.6 目標設定フローの視覚化
体重目標をスライダー+目盛りで選ばせ、ペースもスライダー+「標準的なペース(オススメの範囲)」という推奨帯のハイライト付きで選ばせる。選択に応じて「1週間あたり-0.4kg・1ヶ月あたり-1.6kg」がリアルタイムに表示される。現行のアラゴン式計算カード(数値入力→ボタン押下→結果表示)より、スライダーで動かしながら結果を確認できる体験が優れている。

### 5.7 週間プログラム一覧画面
7列(曜日)×カロリー(青いバッジ)/P/F/C(色付きブロック)を1画面に並べ、「今週の目標値」を一望できる。現行の「現在の目標設定」はテキスト1〜2行のコンパクト表示(L-2の決定)だが、この一覧性の高さも選択肢として比較検討する価値がある。

## 6. Fable5に相談したいこと

### 6.0 スコープの前提確認(最優先)
0. **この相談はアプリ全体(会員側・管理者側の両方、ダイエット以外の予約・ホーム・設定画面も含む)のデザイン言語についての相談であり、ダイエットモジュールはその第一弾の適用先に過ぎない、という理解で回答してよいか。** 以降の論点への回答はすべて「reserve7全体にどう適用するか」を前提に判断してほしい。ダイエット週間パネル固有の話に閉じた回答が必要な場合は、そのようにこちらから明示する。

### 6.1 1つ目の参考画像(黒ベース・達成度色)に関する論点
1. **配色方針の転換**: N-4で確定した単一アクセントカラー(blue)を維持しつつ「達成度を示す状態色」として緑/オレンジを別レイヤーで追加するのか、それとも週間パネル画面に限りN-4を上書きして緑/オレンジを主役にするのか
2. **評価方針の転換**: A-4/H-2で確定した「パネルは善悪を判断しない、色は中立」という原則を、この画面に限り撤回して「目標超過は警告色で示す」方針に変えてよいか。変える場合、会員のホーム画面(A-4が生きている場所)には波及させないという理解でよいか
3. **項目識別色(PFC等)と達成度色の同居**: 参考画像のように達成度色一本に統一し、P/F/C等の項目識別色(amber/emerald/blue)は廃止するのか、それとも両方を何らかの形で残すのか
4. **タイポグラフィのメリハリ**: 実績値を現状よりかなり大きく・太くする(参考画像同等)という方向性への賛否。N-4で「見出し・主数値=font-semibold」と決めたが、参考画像はそれよりもう一段強いサイズ差・太さを使っている
5. **バーの表現**: 単色の連続塗り(現行のWeeklySummaryPanel総合モードの実装)から、緑→黄→オレンジのグラデーション塗りに変更するか

### 6.2 2つ目の参考画像(MyFitnessPal)から取り入れる要素
6. **レイアウト形式**: 現行の縦積み4行リスト(H-2で確定)を、5.1のような2列カードグリッドに作り替えるか。作り替える場合、H-2で確定した「記録日平均・前週比・7ブロック・週合計」という情報量を2列カード内にどう収めるか(情報を削るか、カードを大きくするか)
7. **色分けの一貫性強化**: 現行のP=amber/F=emerald/C=blueをアプリ全体でより一貫させる(5.2)。特にカロリーの色を何にするか(現行は項目識別色を持たず、達成度%表示のみ)
8. **ダッシュボードのミニプレビュー(5.3)・記録継続の30日ドット(5.5)**: これらを新規追加する価値があるか。追加する場合、どの画面に置くか(ホーム/分析タブ/体重タブ)
9. **目標設定のスライダーUI(5.6)**: アラゴン式カロリー計算カードの入力を、現行の数値入力からスライダー+推奨範囲ハイライトに作り替えるか
10. **適用範囲とロールアウト順序**: 週間パネルへの適用を皮切りに、体重タブ・カロリー設定タブ等の他のダイエット関連画面、さらに予約タブ・ホーム・設定画面など会員側・管理者側のアプリ全体へ、どういう順序で展開していくか(N-7のPR-N4以降の計画を、今回の結論を踏まえて更新する形で提案してほしい)
