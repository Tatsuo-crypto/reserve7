## 2. カロミル・あすけん等と比べて足りないもの

### 2.1 追いかけるべき差（UIの作法 — これは全部直せる）

1. **1画面1メッセージ**: プロのアプリは1画面で言いたいことが1つ。reserve7はまだ1画面に情報を詰めがち（padding値の細かさが16種類あるのは、画面ごとに「詰め込み調整」をした痕跡）
2. **タップ領域の大きさ**: プロは押せる物が全部大きい（最低44px）。`text-[10px]`が114箇所ある=小さすぎる文字と小さすぎるボタンが多数
3. **空状態のデザイン**: データがゼロの時（記録なし・予約なし）に、プロは「イラスト＋次にやること1つ」を見せる。reserve7は空白か素っ気ない1行。**新規会員が最初に見るのは全部空状態**なので、第一印象＝空状態の質
4. **待ち時間の見せ方**: プロはスケルトン（灰色のプレースホルダー）でチラつきを防ぐ。reserve7はほぼ未実装（2ファイル）
5. **文言のトーン**: プロは「エラー: 保存に失敗しました」ではなく「保存できませんでした。もう一度お試しください」。機械の言葉と人の言葉の差

### 2.2 追いかけなくていい差（あちらの土俵 — 戦わない）

- **食品データベース検索・バーコード読み取り**: あすけん/カロミルの本丸。数十万件のDB運営が必要で、10社販売の規模では永遠に勝てない。**reserve7は「写真/スクショをAIが読む＋トレーナーが見てくれる」で戦う** — あちらに無いのは「人間のトレーナーが毎週見てくれる」ことで、それがこのアプリの正体
- キャラクター・ゲーミフィケーション・広告モデル: B2C大量ユーザー向けの装備。不要

## 3. 処方箋 — 「一発でバシッと」の中身（統一仕様）

やることは新しい決定ではなく、**既決事項の完遂＋仕様を4つの表に固定**すること。

### 3.1 文字は4サイズだけ（+例外1）

| 役割 | サイズ | 太さ |
|---|---|---|
| 主数値（体重・カロリー等） | text-3xl | bold + tabular-nums |
| 見出し・ボタン | text-base | semibold |
| 本文 | text-sm | normal |
| 補足・ラベル | text-xs | normal + text-secondary |

`text-[10px]`以下は**全廃**（読めない文字は無いのと同じ）。16種類→4種類へ。

### 3.2 ボタンは3サイズ・4色だけ

高さ: 大48px（画面の主ボタン）/ 中40px（通常）/ 小32px（インライン）。バリアント: primary（オレンジ）/ secondary / ghost / destructive。**226箇所の生buttonを全部これに置き換える** — これが「バシッと感」の8割を作る。

### 3.3 角丸3種・余白は4の倍数

角丸: full（バッジ・ピル）/ lg（ボタン・入力）/ 2xl（カード）のみ。md・無印・3xl・smは機械置換で排除。余白はp-2/3/4/6のみに整理。

### 3.4 画面の型3つ

すべての画面を「一覧型（リスト＋空状態）」「詳細型（主数値→グラフ→操作）」「入力型（1画面1タスク）」のどれかに当てはめる。型に合わない画面＝情報が入りすぎている画面。
